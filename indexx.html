Stage 01 · Freewheel
Freewheel Fetcher
Poll Freewheel API for PENDING political ad creatives updated since last run
What it fetches
Endpoint: GET /v1/creative/programmatic_creatives
Filter: approval_status = PENDING (programmatic creatives only)
Political filter: industries[ ].name contains "politic" (case-insensitive)
Paginate 100 records per page until all results retrieved
Freewheel data model notes
approval_status is a programmatic-only field — not available on advertiser module creatives
In the advertiser module, a creative assigned to an active placement implies approval (no explicit approval field)
Brand is set at the IO (Insertion Order) level in the advertiser module hierarchy
Industry is located in a different place than Brand in the advertiser module — depends on account setup

Stage 02 · VAST XML
VAST XML Parser
Fetch and parse each creative's VAST XML to extract metadata not available in the Freewheel API response
Fields extracted from VAST XML
adDescription — from <AdTitle> element
fileFormat — from MediaFile type attribute (e.g. MP4)
fileSize — from MediaFile fileSize attribute, converted bytes → MB
Error handling
VAST URL cached per creative in-run to avoid duplicate fetches
If VAST fetch fails: deliver creative with null fileFormat / fileSize — do not skip
Failures logged in run summary as vast_parse_errors count
creatives enriched
3
Stage 03 · IronDome
IronDome Scanner
Submit each creative to IronDome for spec validation — builds the flagged creatives list
Scan logic
Send creative_id + vast_url to IronDome scan endpoint
Returns PASS or FAIL per creative + feedback string
FAIL → creative added to flagged list, feedback → GeneralNotes field
PASS → creative added to full list only
Error handling
3 retries with exponential backoff (5s, 10s, 20s)
Unreachable after retries → scan_status = UNKNOWN · full list only
Alert triggered if >10% of creatives fail scan
scan results collected
4
Stage 04 · ComplyAI
ComplyAI Formatter & Sender
Map all enriched fields to the ComplyAI JSON schema · validate · deliver two payloads
Field mapping (required)
adId · BusinessArea (hardcoded: Streaming) · adType (Political)
adDescription · fileFormat · fileSize — from VAST XML
duration · industry · approvalStatus · approvalNotes — from Freewheel
advertiserId · advertiserName · brandProduct — from Freewheel brands[ ]
GeneralNotes — IronDome feedback string (optional)
Delivery
Payload 1: full political ads list (all PENDING political creatives)
Payload 2: flagged creatives list (IronDome FAIL records only)
Pydantic schema validation before delivery — failures → dead-letter queue
3 retries with exponential backoff on delivery
both payloads delivered
5
Stage 05 · ComplyAI Review
ComplyAI Compliance Review
Compliance team reviews both payloads against regulatory requirements and internal policy
Reviewes
Full political ads list — all PENDING creatives reviewed for compliance
Flagged creatives list — IronDome-flagged items prioritised for review
GeneralNotes (IronDome feedback) surfaced per flagged creative
Outcome
Compliance determination made per creative: compliant or non-compliant
Non-compliant or uncertain creatives escalated to Ad Ops team for final decision

Step 6 · Ad Ops — Approve or Reject
Ad Ops team receives escalation from ComplyAI and makes the final creative disposition decision
✓ Approve
Creative passes compliance review. Ad Ops approves for trafficking. Creative proceeds to go live on schedule.
✗ Reject
Creative fails compliance review. Ad Ops rejects. Creative is held from going live. DSP / advertiser notified for remediation.
